## 4. 构建步骤

### 4.1 环境变量

```bash
export QAIRT_PATH=D:/dev/qairt/2.39.0.250926
export ANDROID_NDK_ROOT=D:/dev/android_sdk/ndk/28.2.13676358
```

### 4.2 配置 CMake

```bash
cd app/src/main/cpp

# 必须显式指定 ninja 路径（Windows 下 cmake 可能找到错误的版本）
D:/dev/android_sdk/cmake/3.22.1/bin/cmake.exe --preset android-release \
    -DCMAKE_C_COMPILER_LAUNCHER="" \
    -DCMAKE_CXX_COMPILER_LAUNCHER="" \
    -DCMAKE_MAKE_PROGRAM=D:/dev/android_sdk/cmake/3.22.1/bin/ninja.exe
```

**关键参数说明**:
- `-DCMAKE_C_COMPILER_LAUNCHER=""` — 禁用 ccache（Windows 下通常不可用）
- `-DCMAKE_MAKE_PROGRAM=...` — 显式指定 ninja 路径

### 4.3 编译

```bash
D:/dev/android_sdk/cmake/3.22.1/bin/cmake.exe --build --preset android-release
```

### 4.4 复制产物

```bash
# 复制推理核心库
cp -f build/android/bin/arm64-v8a/libstable_diffusion_core.so \
    ../../jniLibs/arm64-v8a/

# 复制 QNN 运行时库
cp -rf build/android/qnnlibs/* ../../assets/qnnlibs/
```

## 5. 遇到的问题及修复

### 5.1 ccache 不可用

**症状**: `CreateProcess failed: The system cannot find the file specified.`

**原因**: CMakePresets.json 配置了 `CMAKE_C_COMPILER_LAUNCHER=ccache`，但 Windows 下没有 ccache。

**修复**: 构建时传入 `-DCMAKE_C_COMPILER_LAUNCHER="" -DCMAKE_CXX_COMPILER_LAUNCHER=""`。

### 5.2 SampleApp 代码版本不匹配

**症状**:
```
error: use of undeclared identifier 'QNN_TENSOR_DATA_FORMAT_UBWC_NV124R'
error: use of undeclared identifier 'QNN_DATATYPE_BFLOAT_16'
error: too few arguments to function call, expected 2, have 1
```

**原因**: `3rdparty/SampleApp/` 目录中的代码是 QNN 2.45 版本的，包含了 2.45 新增的 API 和枚举。

**修复**:
```bash
# 1. 删除旧的 SampleApp
rm -rf 3rdparty/SampleApp

# 2. 从 QNN 2.39 SDK 复制
cp -r D:/dev/qairt/2.39.0.250926/examples/QNN/SampleApp/SampleApp \
    3rdparty/SampleApp

# 3. 应用项目自定义的 patch
cd app/src/main/cpp
git show master:app/src/main/cpp/SampleApp.patch > /tmp/SampleApp.patch
patch -p1 < /tmp/SampleApp.patch
```

**SampleApp.patch 的作用**:
- 添加 `fcntl.h` / `sys/mman.h` 等头文件
- 注释掉 `initialize()` 中的输出目录创建和输入文件列表读取
- 将 `contextCreateFromBinary` 改为 mmap 方式读取（降低内存占用）

### 5.3 Rust 1.96 autoref lint 错误

**症状**:
```
error: implicit autoref creates a reference to the dereference of a raw pointer
--> src\lib.rs:218
    *out_len = (*handle).decode_str.len();
```

**原因**: Rust 1.96 将 `dangerous_implicit_autorefs` lint 升级为 error。

**修复**: 修改 `3rdparty/tokenizers-cpp/rust/src/lib.rs`，显式取引用：
```rust
// 修改前 (line 218)
*out_len = (*handle).decode_str.len();
// 修改后
*out_len = (&(*handle).decode_str).len();

// 修改前 (line 251)
*out_len = (*handle).id_to_token_result.len();
// 修改后
*out_len = (&(*handle).id_to_token_result).len();
```

### 5.4 sentencepiece 子模块未初始化

**症状**:
```
fatal error: 'third_party/absl/strings/string_view.h' file not found
```

**原因**: tokenizers-cpp 的 sentencepiece 子模块未初始化，导致 abseil 头文件缺失。

**修复**:
```bash
cd app/src/main/cpp/3rdparty/tokenizers-cpp
git submodule update --init --recursive
```

### 5.5 CMakePresets.json NDK 路径错误

**症状**:
```
Could not find toolchain file: /data/android-ndk-r28/build/cmake/android.toolchain.cmake
```

**原因**: CMakePresets.json 的 `environment` 节硬编码了 Linux 路径。

**修复**: 修改 `CMakePresets.json`：
```json
// 修改前
"environment": {
    "ANDROID_NDK_ROOT": "/data/android-ndk-r28"
}

// 修改后
"environment": {
    "ANDROID_NDK_ROOT": "D:/dev/android_sdk/ndk/28.2.13676358"
}
```
